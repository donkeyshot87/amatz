1. סקירה כללית

מערכת ניהול פרויקטים מותאמת אישית למפעל אמץ אלומיניום (12 עובדים). המערכת מספקת נראות מרכזית על כל הפרויקטים, מעקב שלבים עם אחריות ברורה, התראות גבייה אוטומטיות, וניהול "זנבות" (בעיות לאחר מסירה).

מה לא נכלל בגרסה 1: אינטגרציה עם אופרה, שכר, מלאי, הנהלת חשבונות, מובייל, התראות אימייל/וואטסאפ, דיווח שעות.

5. לוגיקה עסקית

5.1 שלבי פרויקט — סדר קבוע

#שלבאחראיאחוז גבייה
1חתימה על הסכםאיציק5%
2שרטוטיםנמרוד5%
3משקוף עיוורסלאח10%
4הזמנת חומרנמרוד40%
5ייצוריעקב + נמרוד0%
6התקנהסלאח30%
7מסירהסלאח10%

בעת יצירת פרויקט — 7 שלבים נוצרים אוטומטית עם owner_id לפי הטבלה.

5.2 כלל גבייה אוטומטי

כשמשתמש מסמן שלב כ-completed:

  1. נרשמת רשומה ב-stage_history
  2. אם billing_pct > 0 — נוצרת רשומה ב-billing_alerts עם amount = contract_value × billing_pct / 100
  3. מירי רואה badge מעודכן בלוח הבקרה (מתרענן בטעינה, לא real-time)
שלב 5 (ייצור) — לא נוצרת התראת גבייה.

5.3 סטטוס פרויקט

  • active — פרויקט בתהליך
  • delivered_with_issues — שלב 7 הושלם אך יש tail_issues פתוחים
  • closed — כל השלבים הושלמו + אין tail_issues פתוחים → מעבר אוטומטי

6. מסכים

6.1 לוח בקרה ראשי (כולם)
כרטיסיות פרויקטים עם צבע סטטוס · פרויקטים עם זנב פתוח בראש הרשימה · פילטרים: סטטוס, אחראי, תאריך מסירה · Badge התראות גבייה (מירי/איציק/ניר בלבד) · כפתור "פרויקט חדש"
6.2 דף פרויקט מלא
כותרת: שם + מספר (P-001) + לקוח + תאריך מסירה · ערך חוזה ועלות מוערכת (איציק/ניר בלבד) · ציר זמן 7 שלבים עם accordion להערות וקבצים · כפתור "סיים שלב" לאחראי השלב · קטע זנבות · היסטוריית שינויים (איציק/ניר בלבד)
6.3 מסך גבייה (מירי)
רשימת billing_alerts ממוינת: pending ראשון · לכל התראה: שם פרויקט, שלב, סכום, תאריך, כפתורי "בטיפול" / "טופל" · סיכום: סה"כ ממתין / סה"כ טופל החודש
6.4 מסך ייצור (נמרוד + יעקב)
פרויקטים בשלבים 2, 4, 5 · שלב נוכחי, אחראי, הערות, קישור לקבצי שרטוטים · כפתור עדכון סטטוס שלב
6.5 מסך שטח (סלאח)
פרויקטים שסלאח הוא האחראי הנוכחי · עדכון שלבים 3, 6, 7 · העלאת קבצים · כפתור "פתח בעיית זנב"
6.6 מסך ניהול (איציק + ניר)
כל הפרויקטים כולל ערך חוזה ורווחיות · היסטוריית שינויים מלאה · ניהול משתמשים · סיכום חודשי

8. הנחיות פיתוח

8.1 Row Level Security

CREATE POLICY "finance_only" ON projects
  FOR SELECT USING (
    auth.uid() IN (SELECT id FROM user_profiles WHERE role IN ('admin','developer'))
  );

8.2 שפה וממשק

  • ממשק מלא בעברית, RTL
  • הודעות שגיאה בעברית
  • שדות טכניים (IDs, paths) — אנגלית מותרת

8.3 UX

  • לאחר עדכון סטטוס — confirmation ברור
  • כפתורי פעולה — גלויים רק לבעלי הרשאה
  • טעינה — skeleton screens (לא spinner ריק)
  • Validation בטפסים — בעברית ברורה

8.4 Supabase Storage

  • Bucket: project-attachments
  • Path: {project_id}/{stage_id?}/{file_name}
  • קריאה: משתמשים מחוברים בלבד · כתיבה: לפי RLS
